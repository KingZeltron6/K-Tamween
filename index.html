User Journey

The full flow goes from eligibility login to exploring goods, choosing quantities within quota, booking a slot, and confirming collection or delivery.

1. Login
Enter Civil ID and phone number to access the user account and start the request journey.
2. Browse Goods
Explore categories like basics, meat, canned goods, and healthy products using search and filters.
3. Select Quota
Choose product quantities while staying within the legal monthly allowance shown by the system.
4. Book Method
Pick delivery or branch pickup, then choose a convenient date and time.
5. Confirm
Receive a clear booking confirmation with method, slot, and product summary.

Technology & System Design

The concept is presented as a scalable system using FastAPI for backend performance, React Native (Expo) for cross-platform mobile support, MongoDB for flexible data handling, and strong attention to localization, stability, and secure state handling.

⚙️
FastAPI Backend
Chosen for speed and asynchronous handling of multiple concurrent requests.
📱
React Native (Expo)
One codebase that can run on both iOS and Android as the project grows beyond prototype stage.
🛢️
MongoDB Database
Flexible enough to manage diverse subsidy item data and future supply center updates.
🔐
Security & Localization
Includes secure logout thinking, local state cleaning, and full Arabic / English support.

Future Roadmap
